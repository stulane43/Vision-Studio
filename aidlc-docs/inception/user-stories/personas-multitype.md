# Personas — Multi-Type Vision Documents

> Extends the v1 personas (which centered on a developer/founder turning an idea into a dev vision doc) with the business-user personas this feature targets.

## P1 — Business User (Bex) — *primary new persona*
- **Role:** Non-technical knowledge worker (ops, marketing, comms, program coordination).
- **Goal:** Turn a vague task ("draft a customer comms plan for a policy change") into clear AI instructions that produce a usable first draft.
- **Pain:** Gives AI tools incomplete prompts; gets generic output needing multiple rewrites.
- **Needs:** A safe default type, guided questions, a document that's both a human brief and a copy-ready AI prompt.
- **Success:** "Ready to use with AI" with at most minor edits.

## P2 — Manager / Team Lead (Mara)
- **Role:** Assigns work to people or AI; needs an executive presentation.
- **Goal:** Produce a presentation brief (e.g., 10-slide executive update with speaker notes + recommendation).
- **Pain:** "Make me a deck" yields unfocused slides; expectations aren't captured.
- **Needs:** Presentation type capturing audience, desired action, key message, slide count, output format.
- **Success:** Fewer revisions because structure/message/deliverables are explicit.

## P3 — Analyst / Operations User (Avi)
- **Role:** Works with data, reporting, process docs.
- **Goal (MVP):** Uses the Business / General AI Task Brief for analysis/reporting requests (dedicated Spreadsheet Brief is Phase 2).
- **Needs:** Inputs, assumptions, deliverable format, review criteria captured clearly.
- **Success:** Briefs that define inputs/outputs precisely, reducing back-and-forth.

## P4 — Existing Vision Studio User (Devin) — *regression guardian*
- **Role:** Founder/developer already using Vision Studio for development vision documents.
- **Goal:** Keep creating Development Vision Documents exactly as before.
- **Need:** The new type step must not disrupt or degrade the existing dev workflow.
- **Success:** No meaningful change; Development Vision Document is the default and behaves identically.

## P5 — Vision Studio Product Owner (Pat) — *stakeholder*
- **Goal:** Broaden adoption beyond dev use cases while protecting document quality.
- **Need:** Usage measurable by type; lightweight feedback to guide future type investment.
- **Success:** ≥30% of new documents are non-dev types within 90 days; ≥75% rated ready-to-use.
