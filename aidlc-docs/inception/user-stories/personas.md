# Personas — AI-DLC Studio

## P1 — Maya, the Idea-Haver / Founder (primary)
- **Who**: Non- or semi-technical founder with product ideas.
- **Goal**: Turn a one-line idea into a credible plan (and ideally a starting build) without knowing how.
- **Pains**: Blank-page paralysis; doesn't know what questions to ask herself; intimidated by dev tooling.
- **Needs**: Friendly guidance, smart clarifying questions, professional artifacts, a delightful non-intimidating UI.
- **Success**: "I described my idea and walked out with a vision, a plan, and a starter app."

## P2 — Devin, the Developer / Technical Builder
- **Who**: Engineer who wants to accelerate planning + scaffolding.
- **Goal**: Skip boilerplate planning; get well-structured docs and a buildable codebase fast.
- **Pains**: Tedious upfront docs; weak foundations from ad-hoc starts.
- **Needs**: Control (inline editing, request-changes), visibility into the methodology, code export, provider/key config.
- **Success**: "It produced solid foundations and a scaffold I could actually run and extend."

## P3 — Priya, the Product Manager / Consultant
- **Who**: PM/consultant producing vision/requirements/stories for stakeholders.
- **Goal**: Generate high-quality, editable, exportable planning artifacts quickly.
- **Pains**: Writing the same structured docs by hand for every initiative.
- **Needs**: Faithful document structures (vision, requirements, stories), inline editing, export/share.
- **Success**: "I generated stakeholder-ready docs in minutes and tailored them inline."

## System Actor — The AI-DLC Co-pilot (not a human persona)
- **Who**: The AI engine driving each stage (Claude by default; pluggable).
- **Responsibilities**: Analyze context, ask clarifying questions, produce artifacts, revise on request, generate code.
- **Constraints**: Faithful to the bundled AI-DLC rule set; treats user input + its own output as untrusted; never leaks secrets.
